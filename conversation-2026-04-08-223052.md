   204→## Key rules
   205→
   206→1. **All IO goes through ports/adapters.** No `invoke()` calls outside adapter files. (Keeps IO testable and swappable.)
   207→2. **Stores are sync and side-effect free.** No async/await, no imports from services/ports/domain. (Prevents hidden coupling and race conditions.)
   208→3. **Services never subscribe to store changes.** They read stores but must not use `$effect`. Store observation belongs in reactors. (Clear separation of "do work" vs "react to changes".)
   209→4. **Reactors are the only persistent store observers that trigger side effects.** They call services or pure runtime utilities; they never write to stores directly. (Single mechanism for reactive side effects.)
   210→5. **User-triggerable behavior uses the action registry.** Components call `action_registry.execute()`, never services directly for side effects. (One dispatch surface for shortcuts, menus, command palette, and UI.)
   211→6. **Components do not import services.** (Enforces the action registry as the only interaction path.)
   212→7. **Services never import UIStore.** UI orchestration belongs in actions. (Keeps services testable without UI coupling.)
   213→8. **No global singletons.** Use Svelte context + composition root. (Testable, predictable lifecycle.)
   214→9. **One truth per concern.** ProseMirror owns live doc content. Domain stores own persisted state. OpStore owns operation status. UIStore owns layout/dialog state. (No competing sources of truth.)
   215→10. **Cross-feature imports go through entrypoints.** Import `$lib/features/<name>`, never deep paths like `$lib/features/<name>/state/...`. (Stable public surfaces, refactor-safe internals.)

### 9. **CRITICAL DATA FLOW PATHS**

**Visual Editor → Store:**
1. User edits in ProseMirror
2. `on_markdown_change()` callback fires (via dirty_state_plugin)
3. EditorService calls `editor_store.set_markdown(note_id, markdown)`
4. TabStore's dirty state syncs via `tab_dirty_sync` reactor

**Source Editor → Store:**
1. User edits in CodeMirror
2. `on_markdown_change()` emitted
3. EditorStore sets `markdown` directly
4. Dirty state synced

**Visual ↔ Source Mode Switch:**
1. User toggles editor_mode in EditorStore
2. If visual→source: current markdown already in store
3. If source→visual: EditorService calls `set_markdown()` to sync visual editor
4. `sync_source_editor_markdown()` handles state consistency
5. Dirty state re-evaluated via dirty_state_plugin

**Tab Switch:**
1. User clicks tab in TabBar
2. TabStore activates tab (sets active_tab_id, open_note)
3. editor_sync reactor detects change
4. `resolve_editor_sync_open()` checks if note_id or buffer_id changed
5. If changed: `editor_service.open_buffer(open_note, restore_policy)`
6. Restore policy: "reuse_cache" (different note) or "fresh" (same note, reload)
7. EditorService calls prosemirror_adapter.start_session()
8. Prosemirror initialized with `parse_markdown(open_note.markdown)`

**Save Flow:**
1. Note modified (is_dirty=true)
2. Autosave or user save triggered
3. EditorService.flush() gets current markdown via `serialize_markdown()`
4. Markdown persisted to disk
5. EditorStore.mark_clean() called (optionally with saved_content param)
6. dirty_state_plugin baseline updated

   263→  apply_ai_output(
   264→    target: "selection" | "full_note",
   265→    output: string,
   266→    selection: EditorSelectionSnapshot | null,
   267→  ): boolean {
   268→    const open_note = this.editor_store.open_note;
   269→    if (!open_note) return false;
   270→
   271→    if (target === "selection") {
   272→      if (this.editor_store.editor_mode === "source") {
   273→        if (!selection || selection.start === null || selection.end === null) {
   274→          return false;
   275→        }
   276→
   277→        const next_markdown = `${open_note.markdown.slice(0, selection.start)}${output}${open_note.markdown.slice(selection.end)}`;
   278→        this.editor_store.set_markdown(
   279→          open_note.meta.id,
   280→          as_markdown_text(next_markdown),
   281→        );
   282→        this.editor_store.set_dirty(open_note.meta.id, true);
   283→        this.editor_store.set_selection(open_note.meta.id, {
   284→          text: output,
   285→          start: selection.start,
   286→          end: selection.start + output.length,
   287→        });
   288→        return true;
   289→      }
   290→
   291→      if (this.session?.replace_selection) {
   292→        this.session.replace_selection(output);
   293→        this.editor_store.set_dirty(open_note.meta.id, true);
   294→        return true;
   295→      }
   296→
   297→      if (this.session) {
   298→        this.session.insert_text_at_cursor(output);
   299→        this.editor_store.set_dirty(open_note.meta.id, true);
   300→        return true;
   301→      }
   302→
   303→      return false;
   304→    }
   305→
   306→    if (this.session && this.editor_store.editor_mode === "visual") {
   307→      this.session.set_markdown(output);
   308→    }
   309→    this.editor_store.set_markdown(open_note.meta.id, as_markdown_text(output));
   310→    this.editor_store.set_dirty(open_note.meta.id, true);
   311→    this.editor_store.set_selection(open_note.meta.id, null);
   312→    return true;
   313→  }

   342→  flush(): EditorFlushResult | null {
   343→    if (!this.active_note) return null;
   344→
   345→    const mode = this.editor_store.editor_mode;
   346→    if (
   347→      (mode === "source" || this.editor_store.split_view) &&
   348→      this.editor_store.source_content_getter !== null
   349→    ) {
   350→      const markdown = this.editor_store.source_content_getter();
   351→      this.editor_store.set_markdown(
   352→        this.active_note.meta.id,
   353→        as_markdown_text(markdown),
   354→      );
   355→    } else if (this.session && mode === "visual") {
   356→      const markdown = this.session.get_markdown();
   357→      this.editor_store.set_markdown(
   358→        this.active_note.meta.id,
   359→        as_markdown_text(markdown),
   360→      );
   361→    }
   362→
   363→    const open_note = this.editor_store.open_note;
   364→    if (!open_note) return null;
   365→
   366→    const normalized_markdown = as_markdown_text(
   367→      normalize_markdown_line_breaks(open_note.markdown),
   368→    );
   369→    if (normalized_markdown !== open_note.markdown) {
   370→      this.editor_store.set_markdown(open_note.meta.id, normalized_markdown);
   371→    }
   372→
   373→    return {
   374→      note_id: this.active_note.meta.id,
   375→      markdown: normalized_markdown,
   376→    };

   536→  private handle_wiki_suggest_query(generation: number, query: string): void {
   537→    if (!this.is_generation_current(generation)) return;
   538→    const search_service = this.search_service;
   539→    if (!search_service) return;
   540→
   541→    void search_service.suggest_wiki_links(query).then((result) => {
   542→      if (!this.is_generation_current(generation)) return;
   543→      if (result.status === "stale") return;
   544→      if (result.status !== "success") {
   545→        this.session?.set_wiki_suggestions?.([]);
   546→        return;
   547→      }
   548→
   549→      this.session?.set_wiki_suggestions?.(
   550→        this.map_wiki_suggestions(result.results),
   551→      );
   552→    });
   553→  }

   555→  private handle_image_suggest_query(generation: number, query: string): void {
   556→    if (!this.is_generation_current(generation)) return;

   558→    if (!assets_port) return;
   559→    const vault_id = this.vault_store.active_vault_id;
   560→    if (!vault_id) return;
   561→
   562→    void assets_port.search_assets(vault_id, query, 20).then((paths) => {
   563→      if (!this.is_generation_current(generation)) return;
   564→      this.session?.set_image_suggestions?.(
   565→        paths.map((p) => ({
   566→          path: p,
   567→          name: p.split("/").at(-1) ?? p,
   568→        })),
   569→      );
   570→    });
   571→  }

   573→  private handle_tag_suggest_query(generation: number, query: string): void {
   574→    if (!this.is_generation_current(generation)) return;
   575→    const tag_port = this.tag_port;
   576→    if (!tag_port) return;
   577→    const vault_id = this.vault_store.active_vault_id;
   578→    if (!vault_id) return;
   579→
   580→    void tag_port.list_all_tags(vault_id).then((tags) => {
   581→      if (!this.is_generation_current(generation)) return;
   582→      const lower_query = query.toLowerCase();
   583→      const filtered = tags.filter((t) =>
   584→        t.tag.toLowerCase().startsWith(lower_query),
   585→      );
   586→      this.session?.set_tag_suggestions?.(filtered);
   587→    });
   588→  }
   589→
   590→  private handle_cite_suggest_query(query: string): void {
   591→    const reference_store = this.reference_store;
   592→    if (!reference_store) return;
   593→
   594→    const all = reference_store.library_items;
   595→    const items: CiteSuggestionItem[] = [];
   596→    for (const item of all) {
   597→      if (query && !match_query(item, query)) continue;
   598→      items.push({
   599→        citekey: item.id,
   600→        title: item.title ?? "",
   601→        authors: format_authors(item.author),
   602→        year: String(extract_year(item) ?? ""),
   603→      });
   604→      if (items.length >= 20) break;
   605→    }
   606→
   607→    this.session?.set_cite_suggestions?.(items);
   608→  }
   609→
   610→  private handle_cite_accept(citekey: string): void {
   611→    const reference_store = this.reference_store;
   612→    if (!reference_store) return;
   613→
   614→    const item = reference_store.library_items.find((i) => i.id === citekey);
   615→    if (!item) return;
   616→
   617→    const markdown = this.get_markdown();
   618→    if (!markdown) return;
   619→
   620→    const updated = sync_reference_to_markdown(markdown, item);
   621→    if (updated !== markdown) {
   622→      this.sync_visual_from_markdown(updated);
   623→    }
   624→  }

   626→  private create_session_events(generation: number): EditorSessionEvents {
   627→    const events: EditorSessionEvents = {
   628→      on_markdown_change: (markdown: string) => {
   629→        this.with_active_note_id(generation, (id) => {
   630→          this.editor_store.set_markdown(id, as_markdown_text(markdown));
   631→        });
   632→      },
   633→      on_dirty_state_change: (is_dirty: boolean) => {
   634→        this.with_active_note_identity(generation, (id, path) => {
   635→          this.editor_store.set_dirty(id, is_draft_note_path(path) || is_dirty);
   636→        });
   637→      },
   638→      on_cursor_change: (cursor: CursorInfo) => {
   639→        this.with_active_note_id(generation, (id) => {
   640→          this.editor_store.set_cursor(id, cursor);
   641→        });
   642→      },
   643→      on_selection_change: (selection: EditorSelectionSnapshot | null) => {
   644→        this.with_active_note_id(generation, (id) => {
   645→          this.editor_store.set_selection(id, selection);
   646→        });
   647→      },
   648→      on_internal_link_click: (
   649→        raw_path: string,
   650→        base_note_path: string,
   651→        source: InternalLinkSource,
   652→      ) => {
   653→        if (!this.is_generation_current(generation)) return;
   654→        this.callbacks.on_internal_link_click(raw_path, base_note_path, source);
   655→      },
   656→      on_external_link_click: (url: string) => {
   657→        if (!this.is_generation_current(generation)) return;
   658→        this.callbacks.on_external_link_click(url);
   659→      },
   660→      on_image_paste_requested: (image: PastedImagePayload) => {
   661→        this.with_active_note_identity(generation, (id, path) => {
   662→          this.callbacks.on_image_paste_requested(id, path, image);
   663→        });
   664→      },
   665→      on_file_drop_requested: (file: PastedImagePayload) => {
   666→        this.with_active_note_identity(generation, (id, path) => {
   667→          this.callbacks.on_file_drop_requested(id, path, file);
   668→        });
   669→      },
   670→    };
   671→
   672→    if (this.search_service) {
   673→      events.on_wiki_suggest_query = (query: string) => {
   674→        this.handle_wiki_suggest_query(generation, query);
   675→      };
   676→    }
   677→
   678→    if (this.assets_port) {
   679→      events.on_image_suggest_query = (query: string) => {
   680→        this.handle_image_suggest_query(generation, query);
   681→      };
   682→    }
   683→
   684→    if (this.tag_port) {
   685→      events.on_tag_suggest_query = (query: string) => {
   686→        this.handle_tag_suggest_query(generation, query);
   687→      };
   688→    }
   689→
   690→    if (this.reference_store) {
   691→      events.on_cite_suggest_query = (query: string) => {
   692→        this.handle_cite_suggest_query(query);
   693→      };
   694→      events.on_cite_accept = (citekey: string) => {
   695→        this.handle_cite_accept(citekey);
   696→      };
   697→    }
   698→
   699→    if (this.outline_store) {
   700→      const outline_store = this.outline_store;
   701→      events.on_outline_change = (headings) => {
   702→        if (!this.is_generation_current(generation)) return;
   703→        const note_path = this.get_active_note_path();
   704→        outline_store.set_headings(headings, note_path ?? undefined);
   705→      };
   706→    }
   707→
   708→    if (this.callbacks.on_markdown_lsp_hover) {
   709→      const hover_cb = this.callbacks.on_markdown_lsp_hover;
   710→      events.on_markdown_lsp_hover = (line: number, character: number) => {
   711→        const note = this.active_note;
   712→        if (
   713→          !note ||
   714→          !this.is_generation_current(generation) ||
   715→          is_draft_note_path(note.meta.path)
   716→        ) {
   717→          return Promise.resolve(null);
   718→        }
   719→        return hover_cb(note.meta.path, line, character);
   720→      };
   721→    }
   722→

## Editor Round-Trip Audit: Data Loss / Corruption Risks

### BUG 1 (Medium-High): Tab switch during source mode writes old content to new note's store slot

**Location**: `note_editor.svelte:99-103`, `note_editor.svelte:146-150`, `source_editor_content.svelte:283-286`

**Problem**: When the user switches tabs while in source mode, the SourceEditor's `{#key open_note.meta.id}` block destroys the old component. During `onDestroy`, if there's a pending debounced `store_timer` (within 50ms of last keystroke), it calls:

```ts
on_markdown_change(get_content()); // old CM content
```

The `on_markdown_change` callback in the parent template references `open_note.meta.id`, which is a `$derived` from the store. By the time `onDestroy` fires, the store already has the **new** note's ID. So old source content is written to the new note's markdown slot.

**Why it matters**: The subsequent `editor_sync.reactor` → `open_buffer()` overwrites the corruption, but there's a brief window where the store holds wrong content. If autosave were to fire during that window (extremely tight, but possible with very short debounce), it could persist the corruption.

**Fix**: Capture `open_note.meta.id` in a local variable when the SourceEditor mounts, and pass it via a non-reactive prop or closure, so `onDestroy` always references the original note ID.

---

### BUG 2 (Medium): Split view toggle destroys ProseMirror undo history

**Location**: `note_editor.svelte:78-119` vs `120-165`

**Problem**: The visual editor (`use:mount_editor`) lives in different Svelte `{#if}` branches for split vs non-split mode. Toggling split view causes Svelte to destroy the old branch and render the new one. This calls `app_editor_unmount` → `editor_service.unmount()` → `teardown_session()`, destroying the entire PM session and its undo history. When the new branch mounts, `app_editor_mount` → `editor_service.mount()` → `recreate_session()` creates a fresh session from the store markdown.

**Impact**: All undo/redo history is lost on split view toggle. User cannot Ctrl+Z back past a split view toggle.

**Fix**: Instead of two separate `use:mount_editor` directives in different DOM branches, use a single persistent editor DOM node that gets reparented, or keep the PM session alive across split view toggles and only re-attach the view.

---

### BUG 3 (Low-Medium): Source→visual mode switch doesn't restore cursor position in visual editor

**Location**: `app_actions.ts:365-374`

**Problem**: When switching visual→source, the action saves the cursor offset via `get_cursor_markdown_offset()`. But when switching source→visual, the action does:

```ts
services.editor.sync_visual_from_markdown(open_note.markdown);
services.editor.set_editable(true);
```

It never calls `set_cursor_from_markdown_offset()` to restore the cursor position. The visual editor cursor jumps to position 0 (or end of document depending on PM's replaceWith behavior).

**Fix**: In the `source` branch of `editor_toggle_mode`, capture the source editor's cursor offset before flush, then after `sync_visual_from_markdown`, call `services.editor.set_cursor_from_markdown_offset(offset)`.

---

### BUG 4 (Low): Round-trip normalization causes phantom dirty state on mode switch

**Location**: `prosemirror_adapter.ts:299-311`, `app_actions.ts:365-380`

**Problem**: When switching source→visual, `sync_visual_from_markdown()` calls `set_markdown()` in the PM adapter. PM parses the markdown into a doc, then the `create_markdown_change_plugin` serializes it back. If `serialize(parse(md)) !== md` (e.g., trailing whitespace trimmed, empty paragraphs stripped), `current_markdown` diverges from `saved_markdown`. The note is flagged dirty even though the user made no semantic changes. This triggers an unnecessary autosave with the normalized (slightly different) content.

**Impact**: Silent content normalization on every mode switch. The markdown file on disk subtly changes formatting each time the user round-trips through visual mode. Not data loss per se, but unexpected content mutation — problematic for git-tracked vaults.

**Fix**: After `sync_visual_from_markdown`, re-serialize from PM and use the PM-canonical form as the `saved_markdown` baseline.

---

### BUG 5 (Low): `EditorStore.reset()` doesn't clear `source_content_getter` and `cm_content_cache`

**Location**: `editor_store.svelte.ts:206-215`

**Problem**: `reset()` nulls out `open_note`, `cursor`, `selection`, etc., but leaves `source_content_getter` and `cm_content_cache` intact. If a vault is closed and reopened, or the editor is reset for other reasons, stale getters or cached content could persist. A stale `source_content_getter` pointing to a destroyed CodeMirror view would return `""` (due to `view?.` optional chaining), potentially causing empty content on flush.

**Fix**: Add `this.source_content_getter = null; this.cm_content_cache.clear();` to `reset()`.

### BUG 6 (Low): `cm_content_cache` is dead code — never read outside EditorStore

**Location**: `editor_store.svelte.ts:25-39`

**Problem**: `cm_content_cache`, `cache_cm_content()`, `get_cm_content_cache()`, and `clear_cm_content_cache()` are defined but never called by any other file. This is dead code. It also represents a design intent that was never completed — the cache was likely meant to persist source editor state across tab switches, similar to PM's `buffer_map`.

**Impact**: No functional issue, but adds cognitive overhead and the cache is never evicted (memory leak if it were used).

**Fix**: Remove the dead code, or implement the caching if needed for source editor tab restoration.

---

### BUG 7 (Low): `suppress_change_echo` is not exception-safe in `apply_markdown_diff` and `replace_doc_undoable`

**Location**: `prosemirror_adapter.ts:570-572`, `prosemirror_adapter.ts:600-602`

**Problem**: Both methods set `suppress_change_echo = true`, dispatch a PM transaction, then set it back to `false`. If `view.dispatch(tr)` throws (e.g., invalid transaction), `suppress_change_echo` stays `true` forever, permanently silencing markdown change notifications. All subsequent edits would be invisible to the store.

```ts
suppress_change_echo = true;
view.dispatch(tr); // if this throws...
suppress_change_echo = false; // ...this never runs
```

**Fix**: Wrap in try/finally:

```ts
suppress_change_echo = true;
try {
  view.dispatch(tr);
} finally {
  suppress_change_echo = false;
}
```
